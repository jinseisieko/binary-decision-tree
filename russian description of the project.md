V - значение вадоваемое деревом
# Главный интерфейс BinDecisTree<D,V>
_Пользователь должен объявлять структуры деревьев решений использую этот тип._
## Краткое описание
Этот интрфейс описывает основные возможности деревьев решений такие как 
1. непосредственное использование (метод decide);
2. проверка на заполненость структуры (isFull);
3. получение глубины дерева (количество уровней дерева минус один. т.е. дерево состоящее из одной ноды имеет глубину ноль);
4. преобразование дерева в ветку (смотреть что такое ветвь ниже в описание интерфеса BinDecisTreeBranch)
5. получение всех возможных ветвей дерева (getAllBranches)
6. получения всех ветвей дерева с заданной глубиной (getAllBranchesWithDepth)

## Данный интерфейс реализуют два класса: 
1. дерево составленной из отдельных нод которые имеют ссылки на сыновей
2. дерево составленое на массиве (каждый сын может быть выражен через уневерсальное преобразование индекса отца)

Обе реализацию имеют абсолютну разную логику но исполняют одни и теже дейсвия
Поэтому каждая имет свои плючы и минусы и некоторые особенности

# интерфейс BinDecisTreeBranch
_Пользователь использует этот тип для передачи группы нод между структурами, например может вязть сабдерево одной структуры и вставить в другую_
Каждое дерево имеет свою реализацию веток.
## Краткое описание
Ветви представляют собой лишь ридонли структуру и не могут быть модифицированны, но для оптимизации ветви хранят непосрественно адреса на использующиеся елемны в реализациях деревьев
Они предоставляют информацию для билдера (смотреть BinDecisTreeBuilder) для вставки ветви в дерево.

# интерфейс BinDecisTreeBuilder
_Все манипуляции над деревом для изменения поведения происходят через использования билдера_
Билдер своего рода позиция в дереве, изначально билдер находиться в корне дерева
Каждый тип древа реализует этот интрфейс внутри своего тела, что позволяет билдеру взимодействовать со приватными полями
Билдер не изменяет дерево, а изменяетет копию дерева с возможность вернуть инстанс измененного дерева
Билдер преоставлят следующие возможности манипуляций с деревом:
1. вставить условие в текущую позицию
2. вставить результат в текущюю позицию в двух варициях (констатное занчение или функцию)
3. перейти в ноду ниже которая является истинным значение выполнением условия отца
4. преейти в ноду ниже которая является ложным значение выполнения условия отца
5. перейти в соседню ноду (брата)
6. прейти в отца (или вернуться назад)
7. очистить текущую позицию
8. очистить всю ветку ниже
9. вставить ветку начиная с текущей позиции
10. преобразовать билдер в ветку из текущей позиции
11. получить текущий уровень позиции в дереве
12. собрать дерево из билдера
