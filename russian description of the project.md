Библиотека предназначена для работы с бинарными деревьями решений. Она предоставляет гибкий и расширяемый интерфейс, позволяющий создавать, изменять и использовать деревья решений в различных контекстах. Основные компоненты библиотеки — это интерфейсы BinDecisTree, BinDecisTreeBranch и BinDecisTreeBuilder, которые обеспечивают чёткое разделение обязанностей: использование дерева, передача его частей и модификация структуры.

Главный интерфейс BinDecisTree<D,V> определяет основное поведение дерева решений. Параметр D обозначает тип данных, поступающих на вход дерева, а V — тип значения, которое дерево возвращает после выполнения. Этот интерфейс должен использоваться пользователем при объявлении структур деревьев. Он предоставляет следующие возможности:
1. Метод decide(data: D): V — выполняет проход по дереву и возвращает результат на основе входных данных.
2. Метод isComplete(): boolean — проверяет, заполнено ли дерево полностью.
3. Метод getDepth(): int — возвращает глубину дерева, понимаемую как количество уровней минус один (дерево из одной ноды имеет глубину 0).
4. Метод toBranch(): BinDecisTreeBranch<D,V> — преобразует всё дерево в объект-ветвь, который можно передавать между структурами.
5. Метод getAllBranches(): List<BinDecisTreeBranch<D,V>> — возвращает все возможные ветви дерева, начиная от корня и заканчивая листовыми узлами.
6. Метод getAllBranchesWithDepth(depth: int): List<BinDecisTreeBranch<D,V>> — возвращает все ветви дерева, имеющие заданную глубину.

Интерфейс BinDecisTree реализуется двумя классами:
1. DynamicBinDecisTree — дерево, построенное из отдельных узлов, связанных ссылками. Подходит для динамических сценариев, где дерево часто изменяется.
2. ArrayBinDecisTree — дерево, реализованное на основе массива, где позиции потомков вычисляются по индексу родителя. Предназначено для высокопроизводительного чтения, когда структура дерева фиксирована.

Обе реализации имеют разную внутреннюю логику, но предоставляют одинаковый внешний интерфейс. Каждая реализация имеет свои преимущества и недостатки, что позволяет выбирать подходящую в зависимости от задачи.

Интерфейс BinDecisTreeBranch<D,V> предназначен для передачи поддеревьев или ветвей между различными структурами. Например, можно взять часть дерева из одного экземпляра и вставить её в другой. Каждая реализация дерева имеет свою собственную реализацию ветки. Ветки являются неизменяемыми (read-only) структурами. Для оптимизации они могут хранить прямые ссылки на элементы исходного дерева, что позволяет быстро восстанавливать их при вставке. Ветки также содержат информацию, необходимую билдеру для вставки в другое дерево.

Интерфейс BinDecisTreeBuilder<D,V> используется для всех операций по изменению дерева. Билдер представляет собой позицию внутри дерева и изначально находится в корне. Каждый тип дерева реализует этот интерфейс внутри себя, что даёт билдеру доступ к приватным полям. Билдер не изменяет исходное дерево, а создаёт его копию с внесёнными изменениями, после чего возвращает новый экземпляр дерева. Это обеспечивает безопасность состояния и позволяет легко откатывать изменения. Билдер предоставляет следующие операции:
1. insertCondition(condition: Predicate<D>) — вставляет условие в текущую позицию.
2. insertOutcome(handler: Function<D,V>) или insertOutcome(value: V) — вставляет результат в текущую позицию, как функцию или константное значение.
3. goToTrueNode() — переходит в узел, соответствующий истинному значению условия родителя.
4. goToFalseNode() — переходит в узел, соответствующий ложному значению условия родителя.
5. goToSiblingNode() — переходит к соседнему узлу (брату).
6. goBack() — возвращается к родительскому узлу.
7. clear() — очищает текущую позицию (удаляет узел).
8. clearSubtree() — очищает всю ветвь ниже текущей позиции.
9. insertBranch(branch: BinDecisTreeBranch<D,V>) — вставляет переданную ветвь, начиная с текущей позиции.
10. toBranch() — преобразует текущую позицию и всё поддерево в объект-ветвь.
11. getLevel() — возвращает текущий уровень (глубину) позиции в дереве.
12. build() — завершает сборку и возвращает новый экземпляр дерева с внесёнными изменениями.
